Markdown은 제목, 목록, 링크와 이미지를 간단한 문법으로 표현할 수 있는 글쓰기 형식입니다. 문서가 일반 텍스트이므로 GitHub에서 변경 이력을 관리하고 n8n으로 자동 생성하기에도 좋습니다.

## 작성 흐름

1. n8n 입력 폼에 주제와 핵심 메모를 입력합니다.
2. AI가 제목, 요약, 태그와 본문 초안을 만듭니다.
3. 초안은 GitHub 작업 브랜치에 `index.md`로 저장합니다.
4. Netlify 미리보기에서 내용을 확인합니다.
5. 승인한 글만 운영 블로그에 발행합니다.

> 자동화의 목적은 글을 대신 결정하는 것이 아니라, 반복 작업을 줄여 작성자가 내용에 집중하도록 돕는 것입니다.

## 글 작성 규칙

- 본문 제목은 `##`부터 사용합니다.
- 폴더와 주소는 영문 소문자와 하이픈으로 만듭니다.
- 이미지는 글과 같은 폴더에 보관합니다.
- 처음 생성되는 글은 `draft: true`로 설정합니다.

이 구조를 사용하면 R과 Pandoc을 설치하지 않아도 Hugo만으로 기존 디자인의 블로그를 계속 운영할 수 있습니다.
